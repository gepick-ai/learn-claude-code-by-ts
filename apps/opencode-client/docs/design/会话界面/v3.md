# 会话界面 v3

**状态**：v3 需求说明（会话列表项删除会话）  
**关联**：[会话界面 v1](./v1.md)（布局与交互）；[会话界面 v2](./v2.md)（助手 Markdown）；[与 opencode-app 配套对接说明](../与-opencode-app-配套对接说明.md)（数据流、API、Zustand、SSE）

---

## 1. 目的与范围

在 **v1 / v2 已约定的整体布局、对话流、Markdown 呈现** 不变的前提下，v3 在**左侧会话历史列表**中增加：

- 用户**将指针移动到某一 session 列表项**（`SessionListItem`）上时，该项除原有标题与选中态外，应**展示删除入口**（建议：**垃圾桶图标按钮**，具体图标以实现为准，如 `lucide-react` 的 `Trash2`）。
- 用户**点击删除**后，**永久删除该会话**（与后端语义一致：会话及其消息等关联数据一并删除），并更新前端列表与当前选中会话状态。

v3 **不**改变右侧 Chat 主区的发送与流式协议；**不**重新定义 Tool / Markdown 规则。

---

## 2. 交互细则

### 2.1 展示时机

- **悬停（hover）**：当指针位于**该列表项可交互区域**内时，显示删除按钮；移出后隐藏。  
  - **键盘/触控**：需保证**可访问性**（见 §7）：至少支持通过 Tab 聚焦到删除按钮，或提供等价的无鼠标操作路径（例如在列表项获得焦点时显示删除按钮）。
- **布局**：删除按钮不应挤掉标题可读性；建议标题 `truncate` 保留，删除按钮为 **shrink-0** 图标按钮，靠行尾对齐。

### 2.2 点击行为

- **删除按钮**：`click` 应 **`stopPropagation`**（或等价手段），避免触发「选中该会话」的整行点击。
- **删除对象**：删除的是**该列表项对应的 `sessionId`**，不限于「当前高亮会话」；用户可删除**非当前**会话，也可删除**当前正在查看**的会话。

### 2.3 删除当前会话后的状态

- 从 `sessions` 列表中移除该 `sessionId`。
- 从 `messagesBySession` 中移除该 id 的缓存（避免内存泄漏与误展示）。
- **若删除的是 `currentSessionId`**：
  - 若列表中仍有其他会话：将 **`currentSessionId`** 设为**合理下一项**（建议：**删除项在列表中的**下一项**，若无下一项则取**上一项**；实现须稳定、可预测）。
  - 若列表已空：将 **`currentSessionId`** 置为 `null`，右侧展示 v1 已约定的空态提示；并清理 `localStorage` 中当前会话引用键（与《配套对接》一致，如 `STORAGE_SESSION_ID`）。
- **若删除的不是当前会话**：仅更新列表与对应消息缓存，**不改变**当前 `currentSessionId` 与右侧展示。

### 2.4 破坏性操作与确认（建议）

- 删除不可恢复。产品可选：
  - **v3 最低实现**：直接请求删除（与后端一致）；或  
  - **增强**：首次点击后弹出简短确认（浏览器 `confirm`、轻量 Modal 等），文案说明「永久删除」。  
- 需求文档**推荐**至少一种**防误触**手段（确认 或 可撤销 toast 二选一）；具体以产品定稿为准。

### 2.5 请求进行中

- 删除请求进行时：该按钮 **disabled** 或展示 loading，避免重复提交；可选在 store 中维护 `deletingSessionId: string | null`。

## 3. 与后端 API 的约定

[`opencode-app`](../与-opencode-app-配套对接说明.md) 已提供：

| 方法 | 路径 | 成功响应 | 说明 |
|------|------|----------|------|
| `DELETE` | `/session/:sessionId` | JSON `true` | 删除会话及关联数据；`404` 等错误按现有错误约定处理 |

前端需在 **`apps/opencode-client/src/session/session-api.ts`** 增加例如 `deleteSession(sessionId: string): Promise<void>`（或返回布尔/错误结构，与现有 `createSession` / `getMessages` 风格一致），对非 `2xx` 抛出或返回可展示错误供 store 写入 `lastError`。

---

## 4. 前端状态与配套（Zustand）

在 **`sessionStore`** 中建议增加：

- **`deleteSession(sessionId: string): Promise<void>`**（名称以实现为准）：  
  - 调用 `deleteSession` API；  
  - 成功后按 §2.3 更新 `sessions`、`messagesBySession`、`currentSessionId` 与 `localStorage`；  
  - 若该会话上仍有进行中的发送 / SSE，应按《配套对接》在**切会话或清理**时已有模式，**关闭或忽略**指向已删会话的流（避免对已删除 `sessionId` 继续写状态）。

**SSE / `session.deleted`**：后端 DAO 在删除后会 `Bus.publish({ type: "session.deleted", ... })`。若前端 EventSource 已订阅此类事件，可在 v3 **可选**同步 prune 列表（多标签页一致性）；**最低实现**可仅依赖 DELETE 成功后的本地更新。

## 5. 设计代码目录（在 v1 基础上增量）

| 位置 | v3 职责 |
|------|---------|
| `session/history/SessionListItem.tsx` | 悬停/聚焦时展示删除按钮；标题与按钮布局；删除点击 `stopPropagation`。 |
| `session/history/SessionHistoryPanel.tsx` | 将 `onDelete` 或 `deleteSession` 从 store 传入列表项。 |
| `session/session-api.ts` | `DELETE /session/:sessionId` 封装。 |
| `session/session-store.ts` | `deleteSession` 动作及 §2.3 状态迁移。 |

**不**因 v3 新建与 `history`、`chat` 同级的第三会话子域目录。

---

## 6. 与 v1 / v2 的边界

- v1：**左侧列表 + 高亮 + 新建 + 点击切换** —— v3 在**同一列表项组件**上叠加删除，不改变分栏结构。  
- v2：助手 Markdown —— 与 v3 正交。  
- v3 **明确不展开**：删除动画、软删除/回收站、服务端审计日志 UI。

---

## 7. 无障碍与国际化（摘要）

- 删除按钮需 **`aria-label`**（如「删除会话」），不可用仅图标无名称。  
- 键盘用户可聚焦删除按钮并 **Enter/Space** 触发（原生 `button` 即可）。  
- 文案后续若 i18n，删除确认与 `aria-label` 一并纳入。
