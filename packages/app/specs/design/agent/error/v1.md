# LLMError 分支处理图（v1）

`LLMError` 是 Agent 模块对 LLM 调用异常的业务错误集合。入口拿到 `unknown` 错误后，先按各具体 `NamedError` 的 `is()` 方法做类型收窄；能识别的错误进入对应业务分支，不能识别的错误统一包装成 `UnknownError`。

```mermaid
flowchart TD
  Input["unknown error<br/>LLM 调用 / streamText / provider 返回"]

  Input --> OutputLength{"OutputLengthError.is(err)?"}
  OutputLength -- yes --> OutputLengthHandle["输出长度超限<br/>标记消息生成失败<br/>提示用户缩短输入或重试"]
  OutputLength -- no --> Aborted{"AbortedError.is(err)?"}

  Aborted -- yes --> AbortedHandle["请求被中止<br/>按用户取消或连接断开处理<br/>不作为 provider 故障重试"]
  Aborted -- no --> Auth{"AuthError.is(err)?"}

  Auth -- yes --> AuthHandle["Provider 鉴权失败<br/>提示检查 API Key / provider 配置<br/>服务端记录 providerID"]
  Auth -- no --> Structured{"StructuredOutputError.is(err)?"}

  Structured -- yes --> StructuredHandle["结构化输出失败<br/>记录 retries 和错误信息<br/>可提示模型输出格式不符合预期"]
  Structured -- no --> Context{"ContextOverflowError.is(err)?"}

  Context -- yes --> ContextHandle["上下文超限<br/>提示缩短会话上下文<br/>可触发裁剪或摘要策略"]
  Context -- no --> API{"APIError.is(err)?"}

  API -- yes --> Retryable{"err.data.isRetryable?"}
  Retryable -- yes --> RetryableHandle["Provider API 可重试错误<br/>按状态码 / retry 策略处理<br/>保留响应元信息用于排查"]
  Retryable -- no --> ApiHandle["Provider API 不可重试错误<br/>返回稳定错误响应<br/>记录 statusCode / responseBody"]

  API -- no --> UnknownKnown{"UnknownError.is(err)?"}
  UnknownKnown -- yes --> UnknownHandle["已包装未知错误<br/>直接序列化返回或写入消息错误状态"]
  UnknownKnown -- no --> WrapUnknown["包装成 UnknownError<br/>保留 message，内部日志记录原始异常"]
```

## 分支约定

- `OutputLengthError`：模型输出达到长度限制，属于生成结果不可完整产出的业务失败。
- `AbortedError`：请求被主动中止，通常来自用户取消、连接断开或 `AbortSignal`。
- `AuthError`：provider 鉴权或配置问题，应该优先提示配置修复，而不是普通重试。
- `StructuredOutputError`：模型返回无法通过结构化解析，`retries` 用于判断是否已经耗尽修复机会。
- `ContextOverflowError`：上下文过长，适合触发裁剪、摘要或提示用户减少上下文。
- `APIError`：provider HTTP/API 层错误，`isRetryable` 决定是否进入重试分支。
- `UnknownError`：兜底错误，避免把非标准异常直接泄露到跨边界协议。
