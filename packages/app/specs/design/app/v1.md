# App 配置与运行 v1

**状态**：v1 需求说明（`@gepick/app` 配置基线）  
**关联**：

- `packages/app/src/index.ts`（服务入口）
- `packages/app/src/agent/model.ts`（模型配置读取）
- `packages/app/src/code/client-dev/workspace-root.ts`（`APP_PROJECT_PATH` 语义）

**定位**：定义 `@gepick/app` 在本地开发与运行时的最小配置约定，统一环境变量来源与关键目录配置，保证 Agent 与 Code 工作区链路可稳定启动。

---

## 1. 设计目标

1. **配置来源单一**：`@gepick/app` 运行时以 `packages/app/.env` 为默认配置来源。
2. **可交付示例明确**：提供 `packages/app/.env-example`，用于新环境快速复制。
3. **工作区路径可配置**：通过 `APP_PROJECT_PATH` 控制 project 工作区根目录；未设置时退回默认值逻辑。
4. **与代码行为一致**：文档约束与 `workspace-root.ts`、`agent/model.ts` 的读取行为保持一致。

---

## 2. 配置项（v1）

| 变量 | 必填 | 说明 |
|------|------|------|
| `MODEL_ID` | 是 | Agent 默认模型 ID。 |
| `ANTHROPIC_BASE_URL` | 是（使用 Anthropic 兼容网关时） | Provider 基础地址。 |
| `ANTHROPIC_API_KEY` | 是（使用 Anthropic 兼容网关时） | Provider 鉴权密钥。 |
| `APP_PROJECT_PATH` | 否 | project 工作区根路径；未设置时由代码默认到 `<monorepo>/.projects`。 |

---

## 3. 目录与文件约定

```text
packages/app/
  .env
  .env-example
  src/
    index.ts
    agent/model.ts
    code/client-dev/workspace-root.ts
```

---

## 4. 运行约定

- 启动服务前应确保 `packages/app/.env` 已配置可用值。
- `MODEL_ID` 与 provider 配置缺失时，Agent 模型初始化会失败。
- `APP_PROJECT_PATH` 可用于将 project 工作区放置到自定义绝对/相对路径。

---

## 5. 验收标准（DoD）

- `packages/app/.env-example` 包含 v1 所需关键变量与简要说明。
- 在 `packages/app/.env` 提供有效配置后，`@gepick/app` 可正常启动并读取模型配置。
- project 工作区路径在设置/未设置 `APP_PROJECT_PATH` 两种场景下行为可预测。
