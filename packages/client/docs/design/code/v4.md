# Code 业务区 v4（client：预览入口 `client/dist`）

**状态**：v4 需求说明（会话宿主 `@gepick/client`）  
**关联**：

- 服务端权威规范：[Code 业务区 v4（app）](../../../../app/docs/design/code/v4.md)（磁盘结构、基准 `package.json`、App 源码 `preview/`·`client-dev/` 拆分 **以 app 文档为准**）。

**定位**：用户 project 的前端工程 **仅存在于工作区 `client/`**；会话 Code 面板 iframe **只加载基于构建产物的预览**，入口相对 project 根为 **`client/dist/index.html`**。**开发服务器（`vite dev`）不属于 v4**，仍走静态 `dist` + v3 预览网关。

---

## 1. 预览入口（相对 project 工作区根）

| 常量 / 约定 | 值 |
|-------------|-----|
| **探测与单文件读取 API 路径** | **`client/dist/index.html`**（与 [`workspace-preview.ts`](../../../src/code/workspace-preview.ts) 中 `WORKSPACE_PREVIEW_ENTRY` 一致） |
| **iframe `src`（预览网关）** | `/project/{projectId}/preview/` + URL 分段编码后的同上路径（见 `buildWorkspacePreviewEntryUrl`） |

无 **`client/dist/index.html`**（尚未 `npm run build`）时：`fetchWorkspacePreviewHtml` 返回 **missing**，面板 **empty**，提示在 **`client/`** 内安装依赖并构建（文案见 store / `workspace-preview`）。

---

## 2. 与 v3 的差异（client 侧）

| 维度 | v3（及更早） | v4 |
|------|----------------|-----|
| 默认入口文件 | 根目录 `index.html` | **`client/dist/index.html`** |
| 工作区含义 | 可单文件 HTML | **全量前端在 `client/`**，由模板 + 构建产出 `dist` |

`code-panel`：优先 **`iframe src`** 指向预览 URL；仅在网关不可达且已拿到文本降级时 **`srcDoc`**（与实现一致）。

---

## 3. 非目标（v4）

- 在宿主内嵌 **Vite dev server** 或 **HMR** 预览。
- 自动替用户执行 **`npm run build`**（仍由 Agent / 用户触发）。
