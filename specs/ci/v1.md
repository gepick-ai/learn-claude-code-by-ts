# CI(v1)

## 设计目标

本文用于确定 `gepick` 在 v1 阶段的 CI 编排顺序。目标是先保证构建正确与可发布，再考虑性能优化（如 Turbo）。

## 产物定义

- `@gepick/core`：项目基础工具库，提供通用能力（例如 `NamedError`）
- `@gepick/sdk`：供前端使用的后端 API SDK，提供 RPC 风格体验的 HTTP 接口封装函数
- `@gepick/client`：Vite `dist` 静态资源
- `@gepick/app`：最终 Docker 镜像（内含 `client dist`，由 `app` static serve）

## 编排主线

- 安装依赖（workspace 根目录）
- 按包依赖顺序构建
    - 构建 `@gepick/core`
    - 构建 `@gepick/sdk`
    - 构建 `@gepick/client`
    - 构建 `@gepick/app` 
- 基于`@gepick/app`构建Docker镜像（同域部署，复制 `client dist` 到静态目录）
- 推送镜像

## Checklist

在进入 CI 前，先在本地完成以下检查，确保编排主线“可执行且可产出”。

- [ ] **依赖安装可执行**：在 workspace 根目录执行安装命令（如 `npm install` / `pnpm install`）成功，无中断错误。
- [ ] **脚本存在性检查**：确认 workspace 根 `package.json` 与各包 `package.json` 中存在主线需要的脚本（至少包含各包 `build`，以及镜像构建/推送所需命令入口）。
- [ ] **构建顺序可执行**：按 `@gepick/core -> @gepick/sdk -> @gepick/client -> @gepick/app` 顺序逐步执行构建命令，全部返回成功状态码。
- [ ] **命令用法正确**：逐条核对实际使用的命令参数、工作目录与文档一致，不依赖 CI 环境特有变量。
- [ ] **`@gepick/core` 产物校验**：确认构建后产物目录存在（如 `dist`）且包含可被下游包消费的文件。
- [ ] **`@gepick/sdk` 产物校验**：确认构建后产物目录存在（如 `dist`）且导出文件完整。
- [ ] **`@gepick/client` 产物校验**：确认生成 Vite `dist`，并包含可用于静态托管的入口文件（如 `index.html` 与静态资源目录）。
- [ ] **`@gepick/app` 产物校验**：确认 app 构建成功，且静态资源集成路径下可找到 client 构建产物。
- [ ] **Docker 镜像构建检查**：基于 `@gepick/app` 执行镜像构建命令成功，镜像可在本地 `docker images` 中查询到。
- [ ] **本地容器冒烟验证**：本地运行镜像并访问关键路径，确认静态资源可访问、服务可启动。
- [ ] **推送前置检查**：确认镜像 tag、仓库地址、登录态正确，再执行推送，避免 CI 阶段才暴露配置问题。
- [ ] **问题回写**：若任一步骤失败，先修正脚本/命令/文档并重新跑通本地 Checklist，再进入 CI。
