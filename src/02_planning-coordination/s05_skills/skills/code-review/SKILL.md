When doing a code review:
1. Prioritize correctness, regressions, and broken assumptions.
2. Look for edge cases, missing validation, and unsafe defaults.
3. Call out missing tests only when they materially increase risk.
4. Keep summaries brief; findings should come first.
5. If no issue is found, say so clearly and mention remaining risks or gaps.

Preferred review structure:
- Findings
- Open questions or assumptions
- Brief summary
