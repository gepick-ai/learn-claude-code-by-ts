---
name: task-planning
description: Break multi-step work into clear, ordered tasks
tags: planning execution workflow
---

When planning a task:
1. Restate the goal in one sentence.
2. Break the work into a short ordered list of concrete steps.
3. Keep each step action-oriented and verifiable.
4. Prefer a small number of meaningful steps over a long noisy list.
5. Re-check whether the plan actually leads to the requested outcome.

Good task items should:
- start with a verb
- describe one unit of work
- be easy to mark pending, in progress, or completed
