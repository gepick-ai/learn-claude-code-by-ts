When writing technical documentation:
1. Start with the purpose before implementation details.
2. Prefer short sections with clear headings.
3. Explain why something exists, not just what it is.
4. Use examples when a concept is easier to understand from concrete input/output.
5. Keep wording simple and avoid repeating the same point in multiple ways.

Good documentation usually includes:
- what it does
- when to use it
- key constraints or trade-offs
- a small example when helpful
