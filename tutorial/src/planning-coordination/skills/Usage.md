## Skills

## 定义成功标准

判断这个 skill 真的在起作用？

### 量化指标

- 在 90% 的相关查询上会触发该技能
    - 如何衡量： 准备 10～20 条「理应触发该技能」的测试查询；记录有多少次是自动加载该技能，有多少次需要显式调用。
- 在 X 次工具调用内完成工作流
    - 如何衡量： 对同一任务分别开启 / 关闭该技能各跑一遍；统计工具调用次数和总 token 消耗并对比。
- 每个工作流 0 次失败的 API 调用
    - 如何衡量： 在测试运行期间查看 MCP 服务器日志；跟踪重试率和错误码。

### 定性指标

- 用户不必反复提示 Claude 下一步该做什么
    - 如何评估：在测试里记录你需要纠正方向或澄清的频率；向内测用户征集反馈。
- 工作流能在用户不纠偏的情况下跑完
    - 如何评估：用同一请求跑 3～5 次，对比输出在结构一致性和质量上的稳定性。
- 不同会话之间结果一致
    - 如何评估：新用户能否在几乎不需要额外指导的情况下，第一次尝试就完成任务？
